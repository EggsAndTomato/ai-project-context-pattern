# A/B 测试发现：知识库式 vs 精简懒加载式

> 边界：本页记一次已完成的对照测试的结论（过程文档已清理）。本方法论的优势证据见此。

## 测试方式
同一假项目（Marknote），两个无上下文子代理，分别按"知识库式"（本方法论）与"精简懒加载式 PD"做 bootstrap + 导航 + 改动任务，按统一指标对比。

## 强项分布
- **知识库式（本方法论）更强**：维护规则显著性、Lint 自愈、专属知识归宿（架构 / bug / 会话）、导航答案完整度、改动后的上下文维护深度。
- **精简懒加载式更强**：基础层体量更小、每次自动载入成本更低、编码任务首轮上下文更干净。

## 关键证据
知识库式代理在改动任务后**主动跑了一次 Lint**（查 index 一致性 / 孤儿页 / 矛盾 / 交叉引用 / AGENTS.md 行数），无人提示——直接证明"CRITICAL 工具动作级规则 + Lint"能提升维护遵循度（支撑 ADR-003）。

## 结论
两种方案互补；本项目选择只保留知识库式，因为它在"知识长期积累 + 维护健壮性"上更优，且通过 index 永驻 + 扁平结构把成本控制在可接受范围。
