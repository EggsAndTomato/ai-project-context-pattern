# ADR-010：index 摘要用无状态描述符（不放可变状态）

> 日期：2026-06-24  状态：已采纳

## 背景
外部 review 一个 dogfooding 试用项目（QAgent）时发现：其 `AGENTS.md` 的 index 里 BUG-001 摘要写着「（未修复）」，但 bug 页早已标「已修复」、log 也记了修复——**index 摘要成了唯一的过时副本**。

根因有二：
1. **index 同步规则是「创建专用」的**：原规则只说"新增页面后追加一行"，甚至强调"仅此情况追加"，等于暗示"仅更新页面时别动 index"。AI 严格照做（改了 bug 页、写了 log、动了相关页），唯独没碰 index——完全符合字面规则，却产出漂移。
2. **摘要复制了页面内的可变状态**：「未修复」是个可变状态，权威源在 bug 页。状态翻转时只改页面，index 里的副本必然过时——是「同一知识多处复制」反模式的一个隐蔽变体。

Lint 检查表里虽有「index 与实际一致」+「过时主张」，但 LLM 倾向把"一致"当**结构**检查（文件在不在）而非**语义**检查（摘要是否还匹配页面状态），故这类语义漂移能潜伏到外部 review 才被发现。Lint 能**治愈**，不能**预防**。

## 选择
index 摘要统一为**无状态描述符**：只描述"这页讲什么"，**禁止**内嵌可变状态（bug 状态、ADR 状态、决策采纳与否）。可变状态只在对应页面里（单一事实源）。

配套：
- index 同步规则从「新增页面后追加」扩展为同时覆盖「改名 / 合并 / 拆分 / 删除 / 主题漂移」；删掉「仅此情况追加」反向暗示。
- 明确「bug 修复 / ADR 状态翻转一般不需要动 index」（因摘要无状态可改）。
- Lint 增加显式语义项：「摘要含状态词 → 漂移信号，改为无状态描述符」。

## 理由
1. **消灭漂移面最根本**：摘要里没有可变状态，状态翻转时就没有需要同步的东西 → 该类漂移结构性归零，不依赖 LLM 每次细心。
2. **符合第七节对策 1**：让"正确动作"（这里：无需动作）成为"零成本默认动作"。
3. **守住单一事实源**：状态只在一处，别处不复制。
4. Lint 仍保留为兜底（对策 3），但现在要兜的残差大幅减少。

## 后果
- index 失去"一眼看哪些 bug 未修 / 哪些 ADR 活跃"的能力。`bugs/` 通常很小，可翻页；若增多，可另立一个 loose roll-up 页（如 `open-items.md`）集中汇总开放项——状态仍只在一处。
- 摘要要写成"主题"而非"状态"，对作者要求略高（需提炼页面主题），但一次性成本。
