# ADR-006：会话按需、总结后归档（非每次必存）

> 日期：2026-06-19  状态：已采纳

## 背景
会话里会产出大量"为什么、试过什么、定了什么"，只留在聊天里 = 下次全丢。但原始会话日志无限增长、与既有产物重复、腐化极快。

## 选择
`sessions/` **按需归档**：用户主动提出，或 LLM 判断重要并提示用户；**归档前必须先总结**（按 sessions 模板），**原始记录不存**。

## 理由
1. 给"按需总结后的会话"一个正式归宿，而不是一刀切禁止归档。
2. 总结 = 蒸馏（主题 / 关键决策 / 结论 / 下一步），价值在蒸馏物，不在聊天原文。
3. 重要决策同时沉淀到 `decisions/`（ADR）和 `log.md`；session 页只留脉络。
4. 按需触发避免体积膨胀。

## 后果
- `sessions/` 增长可控。
- 需要自律 / 提示机制保证"重要会话"不被漏归档（Lint 可补：检测"被频繁引用却无 session / ADR 记录"的话题）。
